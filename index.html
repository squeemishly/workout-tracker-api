Welcome to Workout Tracker API!

The following endpoints are available. All endpoints will return the data as JSON.

Lifts Endpoints:

  • GET /api/v1/lifts - returns all lifts currently in the database
  • GET /api/v1/lifts/:id - returns the lift object with the specific :id you've passed in or 404 if the food is not found
  • POST /api/v1/lifts - allows creating a new lift with the parameters:
    { lift: { name: "lift name" } }
    If the lift is successfully created, the lift will be returned. If the food is not successfully created, a 400 status code will be returned. Name is a required field.
  • PUT /api/v1/lifts/:id - allows one to update an existing lift with the parameters:
    { lift: { name: "lift name" } }
    If the lift is successfully updated (name is a required field), the food item will be returned. If the food is not successfully updated, a 400 status code will be returned.
  • DELETE /api/v1/lifts/:id - deletes the lift with the id passed in. If the lift can't be found, a 404 will be returned.

Bodyareas Endpoints:

  • GET /api/v1/bodyareas - returns all the bodyareas in the database
  • GET /api/v1/bodyareas/:id - returns the bodyarea associated with the id specified by :id or a 404 if the meal is not found
  • POST /api/v1/bodyareas - creates a new bodyarea with the parameters: { bodyarea: { name: "bodyarea name" } }. If the bodyarea is not successfully created, a 400 status code will be returned. The name field is required.
  • PUT /api/v1/bodyareas - Updates an existing bodyarea with the parameters: { bodyarea: { name: "edited name" } }. If the bodyarea is successfully updated, the bodyarea will be returned. If the bodyarea is not successfully updated, a 400 status code will be returned. The name field is required.
  • DELETE /api/v1/meals/:meal_id/foods/:id - removes the bodyarea with the id :id. If the bodyarea cannot be found, a 404 will be returned.
