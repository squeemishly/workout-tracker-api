Welcome to Workout Tracker API!

The following endpoints are available. All endpoints will return the data as JSON.

Lifts Endpoints:

  • GET /api/v1/lifts
    - returns all lifts currently in the database
  • GET /api/v1/lifts/:id
    - returns the lift object with the specific :id you've passed in or 404 if the food is not found
  • POST /api/v1/lifts
    - allows creating a new lift with the parameters:
    { lift: { name: "lift name" } }
    If the lift is successfully created, the lift will be returned. If the food is not successfully created, a 400 status code will be returned. Name is a required field.
  • PUT /api/v1/lifts/:id
    - allows one to update an existing lift with the parameters:
    { lift: { name: "lift name" } }
    If the lift is successfully updated (name is a required field), the food item will be returned. If the food is not successfully updated, a 400 status code will be returned.
  • DELETE /api/v1/lifts/:id
    - deletes the lift with the id passed in. If the lift can't be found, a 404 will be returned.

Bodyareas Endpoints:

  • GET /api/v1/bodyareas
    - returns all the bodyareas in the database
  • GET /api/v1/bodyareas/:id
    - returns the bodyarea associated with the id specified by :id or a 404 if the meal is not found
  • POST /api/v1/bodyareas
    - creates a new bodyarea with the parameters: { bodyarea: { name: "bodyarea name" } }. If the bodyarea is not successfully created, a 400 status code will be returned. The name field is required.
  • PUT /api/v1/bodyareas
    - Updates an existing bodyarea with the parameters: { bodyarea: { name: "edited name" } }. If the bodyarea is successfully updated, the bodyarea will be returned. If the bodyarea is not successfully updated, a 400 status code will be returned. The name field is required.
  • DELETE /api/v1/meals/:meal_id/foods/:id
    - removes the bodyarea with the id :id. If the bodyarea cannot be found, a 404 will be returned.

Bodyarea-lifts Endpoints:

  • GET /api/v1/bodyarea_lifts_by_bodyarea/:bodyarea_id
    - Returns a list of lifts for the bodyarea identified. Returns a 404 if the bodyarea_id doesn't exist
  • GET /api/v1/bodyareas_by_lift/:lift_id
    - Returns a list of bodyareas for an identified lift. Returns a 404 if the lift_id doesn't exist
  • POST /api/v1/bodyareas/:bodyarea_id/lifts/:lift_id
    - creates a new bodyarea_lift using the ids in the parameters. If a bodyarea-lift is successfully created, it returns a 200. If the bodyarea-lift is not successfully created, a 400 status code will be returned. Both ids must be valid for the bodyarea-lift to be created
  • DELETE /api/v1/bodyareas/:bodyarea_id/lifts/:lift_id
    - Deletes the bodyarea-lift with the id passed in. If the bodyarea-lift can't be found, a 404 will be returned. Both ids must be valid for the bodyarea-lift to be created.

Users Endpoints:

  • POST /api/v1/users
    - Creates a new user with the parameters: { name: "user name", email: "user email", password: "user password" }. Returns a 404 if any of those parameters are missing
  • GET /api/v1/users/:user_id
    - Returns a single user with the id specified in the params. An invalid id in the params returns a 404

Login Endpoints:

  • POST /login
    - Creates a user login with the parameters: { email: "user email", password: "user password" }. Adds a randomly generated token to their record in the database. Returns the user information: "id", "name", "email", "token". Returns a 404 if the email isn't in the database. Returns a 404 if the password is incorrect.
  • GET /logout
    - Returns a single user with the id specified in the params. An invalid id in the params returns a 404

Workouts Endpoints:

  • GET /api/v1/users/:user_id/workouts
    - Returns a list of workouts and the associated lifts, sets, & reps for that workout for the identified user.
